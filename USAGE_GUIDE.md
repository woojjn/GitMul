# 🎯 GitFlow 사용 가이드 - Stage/Unstage & Commit

## 📦 새로운 기능 v0.2.0

Stage/Unstage와 커밋 생성 기능이 추가되었습니다!

---

## 🚀 빠른 시작

### 1. 파일 스테이징하기

```
변경 사항 (3)
└─ Unstaged (3)
   ├─ README.md [modified]        [+] ← 이 버튼 클릭
   ├─ index.js [untracked]        [+]
   └─ 한글파일.txt [untracked]     [+]
```

**방법 1: 개별 파일 Stage**
1. Unstaged 섹션에서 파일에 마우스 호버
2. 오른쪽에 나타나는 `+` 버튼 클릭
3. 파일이 Staged 섹션으로 이동

**방법 2: 모두 한번에 Stage**
1. 상단의 "모두 추가" 버튼 클릭
2. 모든 Unstaged 파일이 Staged로 이동

---

### 2. 파일 언스테이징하기

```
변경 사항 (3)
├─ Staged (2)
│  ├─ README.md [modified]       [-] ← 이 버튼 클릭
│  └─ index.js [untracked]       [-]
```

1. Staged 섹션에서 파일에 마우스 호버
2. 오른쪽에 나타나는 `-` 버튼 클릭
3. 파일이 Unstaged 섹션으로 이동

---

### 3. 커밋 생성하기

**Step 1: 커밋 버튼 활성화 확인**
```
상단 툴바:
[레포지토리 열기] 브랜치: main  [커밋 (2)] ← Staged 파일 있으면 표시
```

**Step 2: 커밋 다이얼로그 열기**
- "커밋 (2)" 버튼 클릭
- 다이얼로그 팝업

**Step 3: 커밋 메시지 작성**

```
┌─────────────────────────────────────────┐
│ 커밋 생성                               │
├─────────────────────────────────────────┤
│ 커밋 메시지              2개 파일 staged │
│ ┌─────────────────────────────────────┐ │
│ │ feat: 사용자 인증 기능 추가          │ │
│ │                                     │ │
│ │ - JWT 토큰 기반 인증 구현           │ │
│ │ - 로그인/로그아웃 API               │ │
│ │ - 세션 관리 추가                    │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ 💡 팁: 명확하고 간결하게 작성하세요     │
│                                         │
│ 커밋 메시지 컨벤션:                     │
│ • feat: 새로운 기능                     │
│ • fix: 버그 수정                        │
│ • docs: 문서 수정                       │
│ • refactor: 코드 리팩토링               │
│                                         │
│                    [취소]  [커밋]       │
└─────────────────────────────────────────┘
```

**Step 4: 커밋**
- "커밋" 버튼 클릭
- 성공 메시지 표시
- 자동으로 새로고침
  - 커밋 히스토리에 새 커밋 추가
  - 변경사항 목록 비워짐

## 🎨 UI 가이드

### 파일 상태별 색상

| 상태 | 색상 | 아이콘 |
|-----|------|--------|
| **Staged** | 🟢 초록색 | ✅ |
| **Modified** | 🟡 노란색 | 📝 |
| **Untracked** | ⚪ 회색 | ➕ |
| **Deleted** | 🔴 빨간색 | ❌ |

### 버튼 가이드

| 버튼 | 위치 | 기능 |
|-----|------|------|
| `+` | Unstaged 파일 호버 | 파일 Stage |
| `-` | Staged 파일 호버 | 파일 Unstage |
| `모두 추가` | 변경사항 헤더 | 모든 파일 Stage |
| `커밋 (N)` | 상단 툴바 | 커밋 다이얼로그 열기 |
| `🔄` | 각 섹션 헤더 | 새로고침 |

## 💡 실전 워크플로우

### 시나리오 1: 기본 커밋

```bash
# 1. 파일 수정
echo "new code" >> index.js

# 2. GitFlow에서 확인
# - Unstaged에 index.js 표시됨

# 3. Stage
# - index.js에 호버 → + 클릭

# 4. 커밋
# - "커밋 (1)" 버튼 클릭
# - 메시지 입력: "fix: 버그 수정"
# - "커밋" 클릭

# 5. 완료!
# - 커밋 히스토리에서 새 커밋 확인
```

### 시나리오 2: 선택적 커밋

```bash
# 상황: 3개 파일 수정했지만 2개만 커밋하고 싶음

# 파일 목록:
# - README.md (문서)
# - index.js (기능 A)
# - utils.js (기능 B, 아직 미완성)

# 작업:
# 1. README.md → + 클릭 (Stage)
# 2. index.js → + 클릭 (Stage)
# 3. utils.js → Stage 안 함 (Unstaged 유지)

# 4. "커밋 (2)" 클릭
# 5. 메시지: "feat: 기능 A 완료"
# 6. 커밋!

# 결과:
# - README.md, index.js만 커밋됨
# - utils.js는 여전히 Unstaged (나중에 커밋 가능)
```

### 시나리오 3: 실수로 Stage했을 때

```bash
# 실수로 잘못된 파일을 Stage함

# 1. Staged 섹션에서 파일 확인
# 2. 잘못된 파일에 호버 → - 클릭
# 3. Unstaged로 돌아감
# 4. 올바른 파일만 다시 Stage
```

## 🐛 문제 해결

### Q: Stage 버튼이 안 보여요
**A:** 파일에 마우스를 올려보세요. 호버 시에만 표시됩니다.

### Q: "커밋" 버튼이 비활성화되어 있어요
**A:** Staged 파일이 없으면 비활성화됩니다. 먼저 파일을 Stage하세요.

### Q: 커밋 후에도 파일이 남아있어요
**A:** Unstaged 파일은 커밋되지 않습니다. Stage하지 않은 파일만 남습니다.

### Q: 한글 파일명이 깨져요
**A:** GitFlow는 자동으로 UTF-8 설정을 적용합니다. 
      만약 문제가 있다면 수동으로:
```bash
git config --global core.quotepath false
```
